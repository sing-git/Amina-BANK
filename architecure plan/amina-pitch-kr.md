# AMINA 발표 가이드 (피치) — 한국어

> 심사 배점: AI 품질 25 · 비용효율 20 · UX/설명가능성 20 · 컴플라이언스 20 · 엔지니어링 15

## 0. 한 줄 (엘리베이터 피치)
**"고객이 온보딩 때 약속한 모습과 실제 행동이 슬금슬금 달라지는 것(KYC 드리프트)을, 공개 신호(뉴스·제재·등기) + 내부 KYC를 합쳐 실시간으로 잡아내는 시스템. 싼 필터로 거르고 비싼 AI는 위험한 소수에만 — 그리고 모든 판단은 사람이 승인합니다."**

---

## 1. 꼭 크게 강조할 것 (Stress out loud) — 우리의 차별점

1. **KYC 드리프트 = 핵심.** 단발성 사기가 아니라 *서서히* 위험해지는 변화를 잡는다. (챌린지가 콕 집은 것)
2. **비용 계층 (cost tiering).** 공짜 필터(규칙·임베딩) → 싼 LLM은 의심 케이스만 → 제일 비싼 모델은 HIGH만.
   → **"1,000건 분석에 약 $2, 그마저도 로컬 모델로 $0 가능"**. (비용효율 20%에 직격)
3. **정책 파일 하나로 확장.** `riskPolicy.json` 하나만 바꾸면 다른 은행에 그대로. *"AI는 정책을 실행만, 정책은 컴플라이언스가 소유."* (평가자가 요구한 "파라미터만 바꾸면" + 엔지니어링)
4. **사람이 항상 결정 (Human-in-the-loop).** 시그널마다 승인/거부/에스컬레이션 + 모든 결정 감사로그. "Advisory only" 배지. (컴플라이언스 20%)
5. **차별화 한 방 — 배심(Jury) 모델.** 검사 vs 변호가 논쟁하고 판사가 판정. 한쪽 의견이 아니라 양쪽을 검증 → 설명가능성·신뢰성 ↑.
6. **제재 동명이인 방어 (2단계).** 이름만 비슷하면 자동차단 ❌ → 점수+관할 확인, 애매하면 사람 검토 큐.
7. **진짜 데이터로 돈다.** 팀 스크래퍼(뉴스/등기/제재) + EventRegistry 실시간 뉴스. 합성은 Layer 2(내부 KYC)뿐, 명확히 라벨.

## 2. 3분 데모 스크립트 (타이밍)

**0:00–0:30 — 문제 + 한 줄**
"은행은 KYC를 한 번 받지만, 고객은 계속 변합니다. SaaS였던 회사가 크립토 거래소로, 조용하던 계좌가 갑자기 역외로… 이걸 실시간으로 잡는 게 AMINA입니다."

**0:30–1:30 — 라이브 데모 (제일 중요)**
1. 대시보드 → **Team portfolio** 토글 → 실제 10개 기업이 점수와 함께 뜸.
2. **HIGH 기업 클릭** (예: NordPay/Terraform) → 보여줄 것:
   - 시그널 카드 (사업모델 피벗, 역외 송금) + **plain-English 이유** + **출처 링크 클릭 → 실제 뉴스**
   - **배심 패널**: 검사 "위험하다" vs 변호 "정상이다" → 판사 판정
   - **시그널별 승인/거부/에스컬레이션 버튼** (사람이 결정)

**1:30–2:15 — 왜 똑똑하고 싼가**
- "이 뉴스가 위험한지 *판단*은 LLM이, *거르기*는 공짜 임베딩이. 그래서 비싼 모델은 5%한테만."
- `riskPolicy.json` 화면 띄우고: "다른 은행? 이 숫자만 바꾸면 끝. 코드 0줄."

**2:15–2:45 — 컴플라이언스**
- "AMINA는 조언만, 사람이 승인. 모든 결정은 감사로그. 제재는 동명이인 오탐 막으려 2단계."

**2:45–3:00 — 마무리**
- "실시간 공개정보 + 내부 KYC, 비용효율, 설명가능, 사람 통제 — 규제 은행에 필요한 전부. 그리고 완전 무료로도 돌아갑니다."

## 3. 프론트엔드 포커스 (UI 만들 때 집중할 것)

심사는 **UX & 설명가능성 20%**. 화면은 "컴플라이언스 담당자가 바로 쓸 수 있게" 보여야 함.

**꼭 있어야 (우선순위 순):**
1. **알림 큐** — 위험 플래그(색), 점수, **드리프트 전환** 표시 ("low → HIGH ⚠"). 한눈에 위험 보임.
2. **상세 화면 = 설명가능성의 핵심**:
   - 시그널별 **plain-English 이유** (점수만 X)
   - **출처 인용 클릭 → 원문** (신뢰)
   - **배심 패널** (검사/변호/판사) ← 시각적 차별화
3. **시그널별 승인/거부/에스컬레이션/메모** 버튼 (사람 통제 보여줌)
4. **"ADVISORY ONLY" 배지** + **SYNTHETIC** 칩 (가드레일)
5. **감사로그** 탭

**톤:** 차분한 회색, 위험 심각도에만 색. 화려함 ❌, 신뢰감 ⭕. (은행이니까)
**하지 말 것:** 대시보드에 비용 숫자 같은 내부지표 X (피치 슬라이드에만).

## 4. 예상 질문 + 답

| 질문 | 답 |
|---|---|
| 합성 데이터인데 정확도는? | Layer 2(내부)만 합성, 명확히 라벨. 주입한 시나리오가 정답라벨 → `npm run eval` 5/5. 탐지 *엔진*은 실제 뉴스에도 작동(Wirecard 데모). |
| 다른 산업/은행도 되나? | `riskPolicy.json` 교체. 가중치·임계값·아키타입 전부 정책 파일. 코드 변경 0. |
| 비용은? | 1,000건당 ~$2 (Claude), 로컬 모델이면 $0. 비싼 모델은 HIGH 5%만. |
| 환각(hallucination)은? | LLM은 주어진 증거만 사용, 출처 인용 강제. 사람이 최종 승인. 키 없으면 결정론적 stub. |
| 가중치 근거? | FATF/Basel 위험요소 tier에서 출발(실소유주·PEP 고위험). 미래엔 실데이터로 회귀학습. *정직하게: 지금은 FATF 기반 + 수동 설정.* |

## 5. 정직하게 (과장 금지)
- "funding(Crunchbase) 연동은 아키텍처상 지원, 라이브는 미연결" — 인정.
- "weights는 FATF 기반 + 튜닝 가능, 실데이터 캘리브레이션은 로드맵" — 인정.
- 리서치는 *영감(inspiration)*, 그대로 복사 아님.
→ 심사위원은 정직함을 좋아함. 과장보다 "여기까지 했고, 여기는 로드맵" 이 신뢰감 줌.
